# 港股本地持倉管理系統 - 需求規格說明書 (Requirement Specification)

> ⚠️ 本文檔記錄原始需求意圖，保持歷史上下文。實現細節請參見 [architecture.md](architecture.md) 和 [tasks.md](tasks.md)。衝突時以 tasks.md 為準。

## 1. 項目概述 (Project Overview)
本項目是一款專為港股市場設計的本地單機版投資組合（Portfolio）管理系統。系統採用 uni-app (Vue3 + TS) 開發，所有資產數據、交易紀錄完全儲存在手機本地 SQLite 數據庫中，無後端伺服器，極端保障隱私與響應速度。界面風格與信息密度參考「招商證券 App」，市場視覺嚴格遵循港股傳統的 **紅漲綠跌**。

## 2. 核心業務邏輯與算法細則 (Business Logic & Edge Cases)

1. **單一貨幣結算**：系統僅支持港幣 (HKD) 結算，不涉及跨市場汇率換算與多幣種風險。
2. **股票代碼與格式化**：
   - 用戶輸入：支援純數字（例如輸入 `700`、`9988`、`3690`）。
   - 本地存儲與顯示：系統自動補全為 5 位等寬字符串（如 `00700`、`09988`、`03690`），並以等寬字體加粗顯示。
   - API 映射：後台向 Yahoo Finance 請求數據時，程序自動將 `00700` 映射為 `0700.HK` 格式。
3. **全市場股票列表與模糊搜索 (Edge Case 1)**：
   - 系統內置一份全港股代碼與名稱的主數據表 `stock_universe`（~2500+ 隻股票），以 JSON 文件打包隨 App 發布，首次啟動時自動導入 SQLite。
   - 所有股票代碼輸入框改為**模糊搜索輸入框**：用戶輸入純數字（≥1 位）後，系統即時查詢本地 `stock_universe` 表（`LIKE '%keyword%'`），在輸入框下方展示匹配的股票代碼與名稱下拉列表（限前 20 條）。
   - 用戶從列表中選中目標股票後，代碼與名稱自動填入表單，無需等待網絡請求。
   - 用戶可通過"同步市場數據"按鈕手動從 Yahoo Finance 拉取最新全市場列表，更新 `stock_universe` 表。
   - 若 `stock_universe` 中未找到目標股票（如新股尚未同步），用戶仍可手動輸入完整代碼並提交（名稱欄將在後續行情刷新時自動補全）。
4. **持倉均價（加權平均成本）演算法**：
   - **買入 (BUY)**：持股數量增加，持倉總成本增加。
     $$\text{新總成本} = \text{原總成本} + (\text{買入單價} \times \text{買入數量}) + \text{交易手續費}$$
     $$\text{新持倉均價} = \frac{\text{新總成本}}{\text{新持股數量}}$$
   - **賣出 (SELL)**：持股數量減少。quantity 存儲**絕對值**（賣出 500 股存 `'500'`），`calculatePositions()` 根據 `type==='SELL'` 執行減法。持倉總成本按「賣出前的持倉均價」等比例減少。**賣出動作絕對不改變當前的持倉均價成本**。
     $$\text{新總成本} = \text{賣出前持倉均價} \times (\text{原持股數量} - \text{賣出數量})$$
5. **清倉與重置機制 (Edge Case 2)**：
   - 當某隻股票因賣出導致持股數量變為 `0`時，該股票狀態變為「已清倉」。
   - 清倉後，該股票過去累積的總成本、均價在持倉列表中**完全抹平重置**。
   - 未來再次買入該股票時，持倉均價**重新從零開始計算**（歷史盈虧結算至大盤總流水，不干涉新倉位成本）。
6. **分紅 (DIVIDEND) 與總資金池機制 (Edge Case 3)**：
   - 錄入分紅時，持股數量變動為 `0`（不影響股數）。
   - **成本除權策略**：分紅淨額（分紅總額 - 扣稅/手續費）直接從該股票的「持倉總成本」中扣除，從而降低持倉均價（允許均價因回本而變為負數）。
   - **總資金池**：系統不設「初始本金充值」功能。首次買入時可用現金允許為負數（代表初始投入）。後續買入、賣出、分紅皆自動在此本地資金池中滾動沖抵。
   - **累計盈虧公式**：$$\text{累計盈虧} = \text{總資產} - \text{淨投入}$$ 其中：$$\text{總資產} = \text{可用現金} + \sum(\text{持倉現價} \times \text{持股數量})$$ $$\text{淨投入} = \sum(\text{買入單價} \times \text{數量} + \text{手續費}) - \sum(\text{賣出單價} \times \text{數量} - \text{手續費}) - \sum(\text{分紅淨額})$$ 累計盈虧為正（紅）代表盈利，為負（綠）代表虧損。
7. **全量行情手動更新機制 (Edge Case 4)**：
   - 系統不在後台自動循環刷行情。
   - 在資產首頁右上角配置一個**「手動刷新」**按鈕，用戶點擊時，App 會**批量抓取所有在 `stocks` 表中有紀錄的股票現價**（包含當前持有以及歷史已清倉的股票）。
   - 這樣做能確保首頁大盤以及「全量流水賬本」頁面中的歷史股票都能呈現最新的市價參考。
   - **重要區分**：首頁大盤的「今日盈虧」**僅計算當前持股數量 > 0 的持倉**；已清倉股票的行情雖被刷新緩存，但其盈虧不計入大盤看板。

## 3. 資料庫表結構設計 (Database Schema)

系統底層使用四大核心本地數據表來驅動所有的金融計賬、行情緩存與全量賬本排序：

### 3.1 股票行情緩存表 (`stocks`)
- `stock_code` (TEXT, 主鍵): 5位純數字字串，如 '00700'。
- `stock_name` (TEXT, 必填): 股票名稱，如 '騰訊控股'。
- `current_price` (TEXT, 預設 '0'): Yahoo Finance 最新市價（字串存儲確保精度）。
- `yesterday_close` (TEXT, 預設 '0'): Yahoo Finance 昨日收盤價（字串存儲確保精度）。
- `updated_at` (TEXT): 行情上次更新時間時間戳 (YYYY-MM-DD HH:MM:SS)。

### 3.2 交易流水賬本表 (`transactions`)
- `id` (INTEGER, 主鍵, 自增)
- `stock_code` (TEXT, 外鍵): 關聯 `stocks.stock_code`。
- `type` (TEXT, 必填): 交易類型，列舉值：`BUY`, `SELL`, `DIVIDEND`。
- `trade_date` (TEXT, 必填): 交易日期 (格式：YYYY-MM-DD)。
- `price` (TEXT, 預設 '0'): 交易單價。買入/賣出時記錄成交單價；分紅時記錄**分紅總額**（非 0）。所有金額以字串存儲確保 big.js 精度閉環。
- `quantity` (TEXT, 預設 '0'): 交易數量。存儲**絕對值**（買入 1000 股存 `'1000'`，賣出 500 股存 `'500'`）。`calculatePositions()` 根據 `type` 判斷加減：BUY → 加上，SELL → 減去。分紅時固定為 `'0'`。支援小數以兼容港股碎股（Odd Lot）交易。
- `fee` (TEXT, 預設 '0'): 相關交易發生的手續費/扣稅。分紅時此欄位記錄紅利代扣稅或手續費。
- `cash_impact` (TEXT, 必填): 該筆交易對可用現金池的淨變動額。以字串存儲確保 big.js 精度閉環。
- `remark` (TEXT): 用戶手動備註。
- `created_at` (TEXT, 必填): 紀錄寫入數據庫的精確系統時間 (YYYY-MM-DD HH:MM:SS)，用於流水賬本排序。

### 3.3 賬戶可用現金表 (`cash_account`)
- `id` (INTEGER, 主鍵, 自增)
- `available_cash` (TEXT, 預設 '0'): 當前賬戶的可用現金（允許為負數，字串存儲確保精度）。
- `updated_at` (TEXT): 資金餘額最後變動時間戳。

### 3.4 全市場股票列表表 (`stock_universe`)
- `stock_code` (TEXT, 主鍵): 5位純數字字串，如 '00700'。
- `stock_name` (TEXT, 必填): 股票官方名稱，如 '騰訊控股'。
- **用途**：全市場港股代碼與名稱的主數據字典（~2500+ 隻），用於交易/分紅錄入頁的模糊搜索輸入提示。以 JSON 文件打包隨 App 發布，首次啟動時自動導入。用戶可通過「同步市場數據」按鈕從 Yahoo Finance 拉取最新列表並覆蓋更新。
- **與 `stocks` 表的職責分離**：`stock_universe` 存全市場代碼+名稱（主數據），`stocks` 僅存用戶交易過的股票+行情緩存（業務數據）。

## 4. 系統頁面地圖與 UI 交互規範 (App Architecture & UI)

系統採用 TabBar 雙分頁主導航結構，配合兩個獨立跳轉的交易錄入頁：

### 4.1 導航頁一：資產持倉大盤 (`pages/index/index.vue`)

#### 看板區
- 看板為深藍漸變卡片（`linear-gradient(135deg, #1e3c72, #2a5298)`），所有文字為白色系，信息分四層級顯示。
- **第一層（最大）**：「總資產 (HKD)」標籤 + 大字數字（56rpx 粗體）。
- **第二層**：左「總市值」+ 右「可用現金」並排（28rpx）。
- **第三層**：三行盈虧指標，每行兩個並排（32rpx）：
  - 「今日盈虧」+「今日收益率」— 紅漲綠跌
  - 「累計盈虧」+「累計收益率」— 紅漲綠跌
  - 「已實現盈虧」+「淨投入」— 字號略小（28rpx），顏色略降級
- **第四層（最小）**：「行情更新：HH:MM:SS」時間戳（18rpx）。
- 右上角常駐兩個操作圖標：📋 本地重算 + 🔄 同步行情。
- 快捷按鈕：【買賣股票】（紅色 `#fa4d56`）與【錄入分紅】（琥珀金 `#f1c40f`），點擊分別跳轉至 trade.vue 和 dividend.vue。
- **可用現金為負數時**：以紅色標註，提示用戶當前處於初始投入階段（尚未回本）。

#### 看板字段定義
- **總資產** = 可用現金 + Σ(持倉現價 × 持股數量)
- **總市值**（新增）= Σ(持倉現價 × 持股數量)，僅 quantity > 0
- **今日盈虧**：僅對 `quantity > 0` 的持倉計算 Σ((current_price - yesterday_close) × quantity)
- **今日收益率**（新增）= 今日盈虧 ÷ 總市值 × 100%
- **累計盈虧** = 總資產 - 淨投入（已實現 + 未實現的整體盈虧）
- **累計收益率**（新增）= 累計盈虧 ÷ 淨投入 × 100%
- **已實現盈虧**：採用總量減法計算。`realizedProfit = totalProfit - unrealizedProfit`
- **淨投入** = Σ(買入總成本) - Σ(賣出總收入) - Σ(分紅淨額)
- **行情更新時間**（新增）：最後一次成功刷新行情的系統時間，顯示格式 `HH:MM:SS`

#### 操作按鈕交互（右上角兩個圖標）
- **📋 本地重算**：觸發 `refreshPortfolioData()`，僅從本地 SQLite 重新計算持倉與盈虧，不拉取行情。適用於下拉刷新手勢（`onPullDownRefresh`）。
- **🔄 同步行情**：批量拉取 Yahoo Finance 行情，更新 `stocks` 表中所有股票的市價與昨收價。
  - **stocks 表為空時**（首次啟動無任何股票）：Toast "暫無股票數據，請先錄入交易"，不發起網路請求。
  - **點擊後**：圖標旋轉動畫 + 按鈕禁用（`isLoading` 鎖，防止重複觸發）。
  - **成功**：圖標停止旋轉 + Toast「行情已刷新」+ 看板與持倉列表即時更新。
  - **失敗**（網路超時 / API 異常）：圖標停止旋轉 + Toast「刷新失敗，請檢查網絡」+ **保留 stocks 表中上次緩存的價格數據不做覆蓋**。本地功能（持倉計算、交易錄入等）不受影響，仍可基於緩存數據正常運行。

#### 持倉列表區
- 僅動態展示目前數量 `> 0` 的非清倉股票。
- **排序規則**：預設按**市值降序**排列（市值最大的股票置頂），市值相同時按代碼升序。用戶可點擊表頭切換排序方式（市值 / 盈虧額 / 佔比）。
- 每行三列三行佈局（2 行主要信息 + 1 行輔助信息），總高約 140rpx：
  - **第一行**：左列等寬大字代碼 + 灰色小字名稱，右列灰色佔比%（`32.5%`）
  - **第二行**（三列）：
    - 左列：現價（28rpx 粗體）+ 漲跌幅%（24rpx）
    - 中列：盈虧額（28rpx 粗體）+ 盈虧%（24rpx）
    - 右列：持股數（28rpx）
  - **第三行**（三列，灰小字 22rpx）：
    - 左列：持倉均價
    - 中列：個股今日盈虧
    - 右列：當前總市值
- **行情顏色規則**：
  - 現價 vs 昨收：`current_price >= yesterday_close` → 紅色，`<` → 綠色（反映當日漲跌）
  - 漲跌幅% 跟隨現價顏色方向
  - 盈虧額/盈虧%：市值 ≥ 除權後總成本 → 紅色（盈利），< 綠色（虧損）；除權後成本 < 0（完全回本）→ 強制紅色
  - 個股今日盈虧跟隨當日漲跌方向
- **佔比計算**（新增）= 個股市值 ÷ 總市值 × 100%，保留一位小數
- **市值格式**：> 100 萬顯示「XXX.X 萬」，> 1 億顯示「X.XX 億」
- **今日盈虧計算**：僅對 `quantity > 0` 的持倉計算（已清倉股票不計入）。
- 單擊列表任意一行 → 平滑喚起「個股歷史流水彈窗」（tx-popup）。

#### 首次啟動空狀態
- 無任何持倉時，看板區仍正常顯示（總資產 = 現金，盈虧 = 0.00）。
- 持倉列表區顯示空狀態引導：
  > 📊
  > 暫無持倉
  > 點擊下方按鈕開始你的第一筆交易

#### 頁面生命週期
- `onShow`：每次頁面顯示時（從其他頁返回 / TabBar 切換）自動調用 `refreshPortfolioData()` 更新全部數據。
- 支持下拉刷新（`onPullDownRefresh`）：手勢下拉觸發 `refreshPortfolioData()`（僅刷新本地計算，不拉取行情）。

### 4.2 導航頁二：全量流水賬本頁 (`pages/history/history.vue`)

#### 功能描述
獨立的分頁賬本。不區分個股，將整個賬戶所有股票發生的交易混在一起按時間線展示。

#### 排序
嚴格按照 `ORDER BY trade_date DESC, created_at DESC` 排序（最新交易在最上方）。

#### 日期分組（新增）
- 流水列表按**月份分組**顯示，每月一個區塊。
- 月份標題行：淺灰藍底色（`#f0f1f5`），顯示「YYYY年M月」+ 右側「共 N 筆」。
- 默認全部展開，可點擊折疊/展開。
- 組內流水按日期倒序排列。

#### 股票篩選器
- 頁面頂部提供股票代碼篩選下拉（`picker` 或橫向 chips），預設選中「全部」。
- 下拉選項動態生成自所有有交易記錄的股票代碼。
- 選中特定股票 → 列表僅顯示該股票的流水。
- 切回「全部」→ 恢復全量流水顯示。

#### 每行流水展示字段（單行緊湊佈局）
每行顯示：交易日期（`MM-DD` 格式）+ 股票代碼 + 股票名稱 + 類型標籤（紅買入 `BUY` / 綠賣出 `SELL` / 金分紅 `DIVIDEND`）+ 數量（買入 `+X`，賣出 `-X`，分紅 `—`）+ 手續費（若 > 0 則灰色小字顯示"費 XX.XX"） + 現金影響金額（買入為負紅色，賣出/分紅為正綠色）
- 若有 `remark`，在行尾以灰色小字顯示；若無則不顯示
- 欄位在手機單行內可容納，緊湊排列；月份信息由上方分組標題提供，行內僅顯示日

#### 左滑刪除完整流程
1. 用戶左滑 → 露出紅色「刪除」按鈕（寬 120rpx，白色字體）
2. 點擊刪除 → `uni.showModal` 二次確認
   - 標題：「刪除確認」
   - 內容：「確定要刪除這筆交易紀錄嗎？這將會重新計算您的持倉成本與可用現金。」
   - 按鈕：「取消」（灰色）/「確認刪除」（紅色 `#fa4d56`）
3. 點擊「確認刪除」→ 調用 `deleteTransaction(id)`
   - 成功 → Toast「已刪除」→ 列表刷新 + 全局狀態刷新
   - 失敗 → Toast「刪除失敗，請重試」→ 列表不刷新
4. 刪除後列表為空 → 顯示空狀態

#### 長按交互
- 長按任意流水行 → 彈出 `uni.showActionSheet`：["刪除", "取消"]
- 選擇「刪除」→ 進入上述二次確認流程

#### 空狀態
- 無任何交易記錄時顯示：
  > 📋
  > 暫無交易記錄
  > 前往持倉頁開始交易

#### 頁面生命週期
- `onShow`：每次頁面顯示時重新查詢全量流水。

### 4.3 獨立頁面：股票買賣錄入頁 (`pages/trade/trade.vue`)

#### 頂部 Tab 切換
- 【買入】/【賣出】雙標籤切換。
- 買入選中時 Tab 底部指示線為紅色（`#fa4d56`），賣出選中時為綠色（`#24a148`）。
- **切換 Tab 時保留已填寫的表單內容**（避免用戶選錯方向後切換回來丟失輸入）。
- 提交按鈕文案跟隨 Tab 變化：「確認買入」（紅色 `#fa4d56`）/「確認賣出」（綠色 `#24a148`）。

#### 股票代碼輸入（模糊搜索）
- 輸入框使用本地 SQLite 模糊搜索：用戶輸入純數字（≥1 位）後，即時查詢 `stock_universe` 表。查詢使用 `WHERE stock_code LIKE '%' || ? || '%'`（任意位置匹配，輸入 `700` 可匹配 `00700`、`01700`、`87000` 等所有含該數字的代碼）。
- 輸入框下方展示匹配的下拉列表（限前 20 條），每行顯示「代碼 + 名稱」。
- 用戶點擊選中後，代碼自動格式化為 5 位等寬字串並填入，名稱同步顯示。
- 若輸入的關鍵字無匹配結果，顯示提示「無匹配股票，可手動輸入代碼提交」。
- 若從持倉彈窗跳轉而來（`?stock_code=00700`），代碼自動填入但**不鎖定**（用戶可自由修改）。

#### 選中股票後的額外資訊
- 選中股票後，自動查詢該股票當前持倉（若有），顯示於名稱下方：
  > "當前持倉：1,000 股 | 持倉均價：320.50"
- 此資訊從 `calculatePositions()` 計算結果中獲取。

#### 賣出時的額外資訊與校驗
- 股票代碼確認後，必須顯示當前持倉資訊（持股數量 + 持倉均價），輔助用戶判斷賣出數量。
- **賣出數量校驗**：提交時若 `賣出數量 > 當前持倉數量` → 阻止提交，Toast "賣出數量超過當前持倉（X 股）"。
- **清倉提示**：若 `賣出數量 === 當前持倉數量`（即賣出後歸零）→ 提交前彈窗確認：「賣出後該股票將清倉，持倉成本將重置。確定繼續？」避免誤操作。

#### 表單字段與校驗規則
| 字段 | 類型 | 預設值 | 校驗規則 | 錯誤提示 |
|------|------|--------|----------|----------|
| 股票代碼 | 模糊搜索 | — | 必須從 `stock_universe` 選中或手動輸入 4-5 位純數字 | "請選擇或輸入有效的港股代碼" |
| 數量 | number | — | 必須 > 0；賣出時 ≤ 持倉數 | "請輸入有效的交易數量" / "賣出數量超過持倉" |
| 單價 | digit | — | 必須 > 0 | "請輸入有效的交易單價" |
| 手續費 | digit | 0 | ≥ 0，可為空（預設 0） | — |
| 交易日期 | date picker | 今天 | 不能晚於今天 | "交易日期不能晚於今天" |
| 備註 | text | — | 可選，最多 200 字 | — |

#### 提交完整流程
1. 點擊提交 → 前端校驗全部字段
2. 校驗通過 → 按鈕變為「提交中...」+ 禁用（`isLoading` 鎖，防止重複提交）
3. 調用 `addTransaction()`
   - **成功** → Toast「買入成功」/「賣出成功」→ 調用 `refreshPortfolioData()` → 800ms 後 `uni.navigateBack()`
   - **失敗** → Toast「提交失敗，請重試」→ 按鈕恢復可點擊
4. 返回首頁後，首頁 `onShow` 自動觸發數據刷新（保證數據一致）

#### 傳參場景（從持倉彈窗 / 分紅頁跳轉）
- `?stock_code=00700`：代碼輸入框自動填入格式化後的代碼並顯示名稱（**預填但不鎖定**，用戶可自由修改以更換股票）。
- 若有 `stock_code` 參數，Tab 預設選中「買入」。

#### 返回保護
- 用戶點擊返回（導航欄返回按鈕或手勢）時：
  - 若表單有任何已填寫內容 → `uni.showModal`：「丟棄當前輸入的內容？」（"繼續編輯" / "丟棄"）
  - 若表單為空 → 直接返回

### 4.4 獨立頁面：分紅錄入頁 (`pages/dividend/dividend.vue`)

### 4.5 核心組件：個股歷史流水彈窗 (`components/tx-popup/tx-popup.vue`)

#### 喚起方式
- 在首頁持倉列表點擊某隻股票 → 從底部半屏平滑滑出。
- 遮罩層（半透明黑色 `rgba(0,0,0,0.5)`）覆蓋背後內容。

#### 關閉方式
- 點擊遮罩層關閉
- 點擊右上角 ✕ 按鈕關閉
- 點擊快捷按鈕後自動關閉（先關彈窗再跳轉）
- （可選）向下滑動手勢關閉

#### 彈窗頭部：股票匯總信息
- 等寬大字代碼 + 股票名稱
- **持倉匯總**（從 `positionList` 中獲取該股票數據）：
  > 持倉 1,000 股 | 持倉均價 320.50
  > 成本盈虧：+15,200.00（紅色）/ -8,300.00（綠色）
- 成本盈虧 = 當前市值 - **除權後持倉總成本**。正為紅（盈利），負為綠（虧損）。

#### 彈窗中部：流水明細列表
- 可滾動區域（`max-height: 55vh`）
- 按 `trade_date DESC, created_at DESC` 倒序展示該股票的歷史交易
- **持倉周期隔離**：`calculatePositions()` 遍歷時額外維護 `positionCycleMap`（`Map<string, number>`），記錄每隻股票當前持倉周期的起始 `transaction.id`。上次清倉（quantity = 0）之前的舊流水不顯示。若再次買入同一股票，彈窗查詢時加 `WHERE id >= cycleStartId`，僅從最新一筆買入開始展示。
- 每行顯示：日期 | 類型標籤 | 數量（買入 +X / 賣出 -X / 分紅 "—"）| 單價（@ XXX.XX，分紅 "—"）| 手續費 | 現金影響金額（頭部已顯示代碼+名稱，行內省略）
- **空狀態**：「暫無交易記錄」（理論上不會出現，因為只有從持倉列表點擊才能開彈窗）
- 數據來源：每次 `visible` 變為 true 時重新查詢該股票的 transactions

#### 彈窗底部：快捷按鈕
- 【去交易】（紅色 `#fa4d56`）→ 關閉彈窗 → `navigateTo` trade.vue 攜帶 `?stock_code=XXXXX`，Tab 預設「買入」
- 【錄入此股分紅】（琥珀金 `#f1c40f`）→ 關閉彈窗 → `navigateTo` dividend.vue 攜帶 `?stock_code=XXXXX`

#### 數據同步
- 彈窗僅查詢展示，不修改數據。
- 若用戶通過快捷按鈕跳轉到交易頁並成功提交 → 返回首頁後，再次點擊同一股票打開彈窗時，應反映最新流水（每次打開重新查詢）。

### 4.6 跨頁面通用行為規範 (Cross-Page UX Standards)

#### 數據一致性保證
```
交易錄入頁 / 分紅頁 提交成功
  → addTransaction() 寫入 SQLite
  → refreshPortfolioData() 更新全局 Ref
  → uni.navigateBack() 返回
  → 首頁 onShow 自動觸發 refreshPortfolioData()
  → 全局 Ref 更新 → 所有綁定頁面自動刷新
```
- 任何寫入操作（交易、分紅、刪除）成功後，下一層頁面必須能立即看到最新數據。
- 彈窗數據在每次 `visible` 變為 true 時重新查詢，確保不展示過期數據。

#### TabBar 切換行為
- 首頁（持倉）↔ 流水頁切換：各自 `onShow` 觸發數據刷新。
- 切換回來時用戶看到最新數據，無需手動刷新。

#### 頁面返回保護（Dirty Form Protection）
- **交易頁 & 分紅頁**：若表單有任何已填寫內容，點擊返回時彈窗確認「丟棄當前輸入的內容？」，按鈕為「繼續編輯」/「丟棄」。表單為空則直接返回。
- **首頁 & 流水頁 & 彈窗**：無需保護（純展示頁面）。

#### 網路異常全局降級
- 任何需要網路的操作（行情刷新、股票名稱查詢）失敗時 → Toast 明確提示用戶。
- **本地操作不受網路狀態影響**：交易錄入、分紅錄入、交易刪除、持倉計算均基於本地 SQLite，完全離線可用。
- 股票名稱 @blur 查詢失敗時：不阻塞交易提交（用戶可手動輸入名稱或直接用代碼提交，名稱欄位會在後續行情刷新時自動更新）。

## 5. 技術與質量清單 (Technical Quality Checklist)
1. **高精度金融計算**：項目內凡涉及金額、均價、手續費、現金流的加減乘除，必須強制使用 `big.js` 庫封裝，嚴禁使用 JavaScript 原生 Floating Number。數據庫中所有金額與數量欄位採用 `TEXT` 類型存儲，與 big.js 形成端到端精度閉環。
2. **雙環境數據庫自適應**：底層 SQLite 連接器具備三環境自適應能力——App Native 使用 `plus.sqlite`，桌面瀏覽器使用 `sql.js`（SQLite WASM + IndexedDB 持久化），Node.js 測試使用 `sql.js` 純內存。三種環境提供相同的 SQL 語法和事務行為。桌面瀏覽器可運行完整系統，用於開發和完整性驗證。
3. **目標平台**：第一版僅支援 5+ App Native 平台（iOS / Android）。桌面瀏覽器作為開發和驗證環境（功能完整），微信小程序不在範圍內（數據庫 API 不相容）。

## 6. 已知限制與設計取捨 (Known Limitations & Design Trade-offs)

以下是本系統第一版明確不支援的場景，記錄於此以供未來迭代參考：

### 6.1 公司行動 (Corporate Actions)
港股常見的公司行動（股票拆分、合股、送紅股、供股、以股代息、私有化等）**第一版不提供專用交易類型**。用戶可通過現有的 BUY / SELL / DIVIDEND 手動模擬（例如拆分後手動修改數量）。未來版本可考慮增加 `SPLIT`、`BONUS` 等交易類型。

### 6.2 數據備份與導出
第一版不內建 JSON/CSV 導出或雲端備份功能。建議用戶通過手機系統級備份（iCloud / Google Backup）保護數據。後續版本可考慮增加導出功能。

### 6.3 交易編輯
第一版僅支援「刪除 + 重建」模式修正錯誤交易。不提供原地編輯功能。這是為了保持流水不可變審計鏈的設計取捨。

### 6.4 全量流水搜索
全量流水賬本頁（history）第一版提供**股票代碼下拉篩選器（picker）**，可篩選特定股票的全部流水。暫不提供全文搜索與日期範圍過濾，後續版本可增加。

### 6.5 多市場/多幣種
系統僅支援港股 + 港幣 (HKD)，不涉及美股、A股或跨幣種匯率換算。這是架構層面的設計取捨，非階段性限制。
